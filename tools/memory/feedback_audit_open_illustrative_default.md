When auditing a chapter, paragraph, or essay against the framework's current state — especially during Stage-3 rigor passes, cross-chapter consistency checks, and apparatus-register checks — default to an open/illustrative reading of framework lists, patterns, inventories, and example sets. The framework's published posture is Ostrom-path: lists are illustrative-with-room-to-grow, not exhaustive enumerations. Treat any closed/canonical reading of the framework as the exception requiring positive evidence (e.g., an explicit ratification of closure; "the [N] mechanisms" framing surviving in current canonical sources after deliberate review).

**Why:** This session (2026-05-12) made the error three times during a Ch 2 Stage-3 Phase A audit. Pattern: (a) declared a "false flag" on Ch 2's "spatial cost severance" usage by treating the terms_index lowercase-current entry as canonical resolution of a conflict with the GuidanceDoc audit's REGRESSED flag, without weighing that the audit was more recent active-review work; (b) when correcting toward a "Phase B framing-consistency check," framed Ch 7's six-patterns observation as a *closed reference frame* to check Ch 2 against — itself the same closure-reading error; (c) the existing Ostrom-path content memory (`reference_ostrom_illustrative_register.md`, ratified 2026-05-02) was already in scope but applied only to content origination, not to audit reading. Author had to teach the same lesson three times in one session. The framework's deliberate posture is open/illustrative; defaulting to closure-readings during audits contaminates the audit verdicts and creates false-positive findings the author has to push back on.

**How to apply:**
- Stage-3 fact-check + voice-polish + audience-load passes against any chapter.
- Cross-chapter consistency audits.
- Apparatus-register and vocabulary-state checks where the question is "does this chapter's framing align with framework state."
- When you see a list of N items in framework documents (6 patterns; 10 commons; 8 cost components; handful of named cases; etc.), read it as illustrative-with-room-to-grow by default. Only treat as closed if an explicit closure-ratification exists.
- When a recent audit flags an apparent regression citing a numbered framework list, check whether the auditor themselves may have made the same closure-reading error before concluding the chapter is wrong.
- Applies equally to bibliography lists, comp-titles sets, example cases, illustrative commons enumerations.
