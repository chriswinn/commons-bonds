# Stage 0 — Publishing-Strategy Rigor Test (Template)

**FIRST ACTION (2026-05-26).** Before any other work, the session firing this template must execute the worktree-isolation step at [`worktree-isolation-paste-text.md`](worktree-isolation-paste-text.md). Branch-contamination root-cause + discipline: [`../memory/feedback_worktree_isolation_for_parallel_sessions.md`](../memory/feedback_worktree_isolation_for_parallel_sessions.md).

**Purpose.** Pre-decision rigor pass that runs BEFORE any drafting begins. Decides whether a proposed publisher-facing piece should exist at all + when it should submit, against book-success criteria.

**When to use.** Substantial publisher-facing derivative work (typically magazine essay; ~3,000+ words; new venue OR potential book-impact). Skip for pieces already cleared in the cascade plan + already-running pipeline.

**When NOT to use.** Book-proposal sections. Internal-scaffolding. Email correspondence. In-book defense paragraphs. Op-eds in cascade-plan-cleared venues.

**Canonical example.** [`publishing/essays/100-barrel/rigor/q1-go-no-go.md`](../../publishing/essays/100-barrel/rigor/q1-go-no-go.md)

---

## How to use this template

1. Fill in the `[BRACKET PLACEHOLDERS]` below with project-specific context.
2. Paste the resulting text into a fresh session.
3. Session reads + produces a rigor-pass artifact with verdict.
4. PM session auto-verifies; author ratifies the verdict.
5. If GO or CONDITIONAL → proceed to Stage 1. If NO-GO → withdraw the workstream.

== Master criterion: the book ==

This rigor pass evaluates the [ARTIFACT NAME]'s net effect on
book-success criteria, NOT its own merits as a standalone publication.
The book is the deliverable; pieces are subordinate. Specifically:

1. **Agent representation by Wave 1** (late July / early August 2026):
   complete book proposal + ≥1 essay at editor-review at a top-tier
   venue + ≥1 named-source interview recorded with quote-permission.
2. **Trade-publisher contract** (target 2026–2027).
3. **Launch positioning + reviews + sales** preserving non-partisan
   framework positioning (FPD v1.0.0 + cascade-plan "Strategic frame"
   point 3).
4. **Framework Positioning Disciplines** (FPD v1.0.0).

A dimension that helps the piece land but hurts a book-success
criterion tilts HURTS. A dimension that dampens piece reception but
strengthens book-success criteria tilts HELPS.

== Question 1 — Does publishing this help or hurt the book? ==

Apply these dimensions explicitly with HELPS and HURTS framings.
Each dimension cuts both ways; the question is which direction the
weight falls.

(a) **Thunder-stealing vs. authority-building.**
    - HURTS: piece reveals framework's load-bearing apparatus in a
      way that makes the book seem redundant.
    - HELPS: piece demonstrates intellectual depth + credibility +
      published-track-record. Builds anticipation for the book.
    - TEST: how much of the framework's load-bearing apparatus lands
      in the piece? <15% = pure-hint = book-helps; 15–30% =
      ambiguous; >30% = thunder-stealing territory.

(b) **Political coding vs. register-positioning.**
    - HURTS: piece codes Commons Bonds as politically partisan.
      Erodes non-partisan framework-positioning strategic asset.
    - HELPS: piece demonstrates framework engages serious intellectual
      debates rather than ducking them.
    - TEST: read source material with a politically-hostile reviewer
      lens. Are partisan flag-points removable without losing
      intellectual content?

(c) **Press-cycle exhaustion vs. pre-warming.**
    - HURTS: venue's reviewer network exhausts engagement on the
      piece rather than the book.
    - HELPS: piece becomes book-launch platform-paragraph ammunition
      + reviewer recognition.
    - TEST: estimate venue's reviewer pool size + overlap with other
      pipeline venues.

(d) **Framework-positioning lock-in vs. stress-test ratification.**
    - HURTS: framing fixed in print → harder to revise during book
      editing.
    - HELPS: framing stress-tested + real reader/reviewer feedback
      collected.
    - TEST: how stable is the framework framing right now? Apparatus
      register ratifications + flagship-term name-defenses landed?

(e) **Blurb-network alienation vs. attraction.**
    - HURTS: side-taking on intellectual debates may antagonize
      blurbers who hold competing positions.
    - HELPS: side-taking attracts blurbers who share positions;
      lukewarm centrism rarely gets blurbs.
    - TEST: enumerate top 5–8 likely-blurber names. For each, what's
      their published position on the piece's load-bearing moves?
      Estimate align-vs-conflict ratio.

(f) **Source-material self-sufficiency** (orthogonal — binary gate).
    Is the source material intellectually load-bearing on its own,
    or did it lean on related material the derivative piece can't
    recover? If self-sufficient → PASS GATE. If not → flag scope-
    expansion required.

(g) **Strategic portfolio value.**
    Does the piece fill the cascade-plan format-mix slot? Or does it
    duplicate territory already covered?

(h) **Cumulative essay-portfolio audience-coverage vs. concentration.**
    The piece does NOT exist in isolation. Test cumulative effect:
    - HELPS coverage-spread: piece adds a distinct audience cell.
    - HURTS audience concentration: piece over-concentrates portfolio
      in a particular register.
    - HURTS cumulative thunder-stealing: cumulative reveal across all
      pieces approaches 30%+ of framework apparatus.

== Verdict synthesis ==

Re-state each dimension's tilt (HELPS / HURTS / NEUTRAL / PASS GATE)
framed explicitly as effect on book-success criteria 1–4.

The verdict synthesizes net direction across all 8 dimensions:

- **GO** — net direction HELPS book-success criteria.
- **NO-GO** — net direction HURTS book-success criteria. Recommend
  withdrawing workstream.
- **CONDITIONAL** — direction is mixed or net-positive only with
  specific scope-adjustments. Specify the conditions and what
  book-success criterion each condition protects.

== Question 2 — Sequencing (if Q1 = GO or CONDITIONAL) ==

Apply these sequencing dimensions:

(a) **Active-pipeline calendar.** Where in the calendar does
    submission slot in without competing for author cognitive
    bandwidth?
(b) **News-peg salience.** Window estimate. Time-sensitive or
    durable?
(c) **Venue cycle.** Response window + publication lead-time.
(d) **Author-bandwidth load.** Ratification gates pending +
    cognitive-load events (interviews, submissions).
(e) **Agent Wave 1 platform impact.** Late July / early August 2026.
    Backward-plan.

Verdict: **FIRE NOW** / **QUEUE FOR LATER** / **DEFER** / **DEFER
INDEFINITELY**.

  tilts framed against book-success criteria
- Q1 verdict (GO / NO-GO / CONDITIONAL)
- Conditions for GO (if CONDITIONAL)
- Q2 sequencing (if GO or CONDITIONAL)
- Risks + mitigations summary
- Recommended next action: which session to fire next + trigger
  condition

== Hard constraints ==

- Do NOT write any portion of the piece itself.
- Do NOT propose meta-conclusions about the v2.0 discipline framework.
- DO surface genuine conflicts with current pipeline; do NOT

- DO consider NO-GO seriously — if the analysis warrants it, return

## Customization notes

- Cumulative-portfolio dimension (h) is only relevant if there are 3+ other essays in pipeline. For a first-essay-in-portfolio, drop dimension (h).
- For non-essay artifacts (e.g., proposal section), reframe dimensions (a)–(h) to fit the artifact type. Most still apply; some (e.g., audience-coverage matrix) don't.
- Add an explicit "CONDITIONAL trigger requires PM verification of items X + Y" if specific conditions need PM-session follow-up before flipping CONDITIONAL → GO.
