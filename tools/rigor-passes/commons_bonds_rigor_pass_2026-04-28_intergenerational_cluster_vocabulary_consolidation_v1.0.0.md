## §3. Methodology — five sub-passes

The rigor pass applies five sub-passes in sequence. Sub-pass output feeds the next.

### §3.1 Sub-pass 1 — Axis-comprehensiveness ratification

**Purpose:** Establish a ratified minimum-spanning set of axes for distinguishing cluster members.

**Procedure:**
1. Inventory candidate axes (~12-13 candidates including author's three).
2. For each candidate axis, test:
   - **Disambiguation value:** does the axis produce a meaningful split for at least 3 cluster members?
   - **Orthogonality:** does the axis distinguish information not captured by already-accepted axes?
   - **Complexity-cost earned:** does the disambiguation buy enough to justify the axis's overhead?
3. Eliminate redundant or low-value axes; promote orthogonal high-value axes.
4. Ratified output: minimum-spanning axis set used as the analytical lens for sub-passes 2, 3, 4.

**Output:** Ratified axis list (likely 5-8 axes).

### §3.2 Sub-pass 2 — Existing-vocabulary-fit per term (Tier A→D ladder)

**Purpose:** Determine for each term whether existing vocabulary suffices, and at what tier — evaluated against the inherited ratified audience (§2.4).

**Procedure (per term):**
1. **Tier A check** — does an established academic-literature term capture the framework's intended meaning *exactly*, AND land cleanly with the B-register reader (Mazzucato/Raworth/Sen profile) without specialist gloss? If yes → adopt verbatim with citation; no coinage.
2. **Tier B check** — if Tier A fails, is there an academic-literature term that's close-but-not-identical, where the framework specializes one or two axes, AND the specialization is decodable to a B-register reader with light footnote? If yes → adopt with explicit specialization footnote.
3. **Tier C check** — if Tier B fails, would descriptive prose adjective+noun (existing English vocabulary) suffice in B-register chapter usage? If yes → no proper-noun framework term; track in terms_index as "preserved compositional phrase" if recurring.
4. **Tier D check** — if Tiers A+B+C all fail, the framework needs to make a distinction no existing vocabulary supports for the B-register reader. Coinage justified — proceed to sub-pass 3.

**Decision rule (under inherited B+D audience):** Default to lowest tier (A → B → C → D) that adequately serves the B-register reader. Coinage (Tier D) is the last resort. For terms in D-modulation chapters where the success-criterion is legal/policy adoption-travel, the Tier verdict additionally tests whether the chosen tier supports regulatory-text migration; failure on that test pushes the verdict to Tier D-with-D-register-specialization (e.g., severed cost as Tier D specifically engineered for legal/policy adoption).

**Strict-fit threshold for Tier A** (ratified default): existing term must capture *every* axis the framework cares about per sub-pass 1 ratified axis list **AND** require no specialist gloss for the B-register reader. Framework specialization on minor axes — OR specialist-gloss requirement — pushes the verdict to Tier B with formal footnote.

**Output:** Per-term Tier verdict (A / B / C / D), with audience-rationale per verdict.

### §3.3 Sub-pass 3 — Mathematical-role audit

**Purpose:** Verify each term's name transparently encodes its mathematical role in the framework.

**Background:** The framework already has implicit role-bearing-suffix discipline:

| Mathematical role | Examples | Suffix convention |
|---|---|---|
| Input variable / Cᵢ | Dynastic Labor Cost, Foreclosure Cost, Externality Tail | usually ends in "Cost" |
| Function inside RCV integrand | Substitutability Function S, Discount Function D | ends in "Function" |
| Primitive on equation's other side | Accountability Bond, Restitution Bond, Foreclosure Bond | ends in "Bond" |
| Output / result | Cost Severance, Severed Cost, IPG | ends in "Severance" or "Gap" |
| Named equation | CS = RCV − B | the equation form itself |
| Decision rule | Asymmetric Regret Rule | ends in "Rule" |
| Methodology | Commons Inversion Test, Four Gates | ends in "Test" or "Gates" |

**Procedure (per term):**
1. Identify the term's mathematical role in the framework.
2. Check: does the term's name transparently encode the role per the convention table?
3. If YES → role-transparent; no flag.
4. If NO → flag for renaming under sub-pass 4 (per-usage may keep it; rename addresses role-ambiguity at the name level).
5. If the role IS ambiguous (term mixes roles in its name, e.g., *"intergenerational cost severance"* mixes Cᵢ-suffix "cost" + output-suffix "severance"), flag for renaming with role-clarifying name candidates.

**Output:** Per-term role-transparency flag (TRANSPARENT / OPAQUE / AMBIGUOUS); rename candidates for non-transparent.

### §3.4 Sub-pass 4 — Per-usage replaceability audit

**Purpose:** Audit each manuscript occurrence of each term for load-bearing-vs-decorative deployment.

**Procedure (per occurrence):**
1. Apply load-bearing-vs-decorative rubric (§3.4.1 below).
2. Assign one of four verdicts:
   - **KEEP** — load-bearing use of named term.
   - **REPLACE-WITH-ACADEMIC** — established academic vocabulary suffices in this context.
   - **REPLACE-WITH-PROSE** — descriptive adjective+noun suffices; named-term-form is decorative. Includes proposed rewrite draft.
   - **ELIMINATE** — sentence rewrites cleaner without the term entirely. Includes proposed rewrite draft.
3. Borderline cases get flagged for author judgment (carry context + my best read of the tradeoff).

**Output:** Per-occurrence verdict matrix; aggregate KEEP/REPLACE/ELIMINATE percentages per term.

#### §3.4.1 Load-bearing-vs-decorative rubric

**Load-bearing patterns (KEEP candidates):**
- *First-introduction-in-chapter* — the term is being introduced for the first time in that chapter; reader needs the name to track the concept.
- *Compressed-scalar-at-argument-climax* — term used as compact handle for an argument climax (e.g., "IPG = 33").
- *Technical-formal context* — Tech Appendix derivation, equation, gates apparatus, glossary definition.
- *Cross-chapter-recurring named handle* — term used as recurring handle the framework relies on across multiple chapters.

**Decorative patterns (REPLACE candidates):**
- *Repeated-within-paragraph* — term appears 2+ times in same paragraph where pronoun or descriptive phrase would suffice.
- *Adjective-form* — term used as adjective ("the dynastic-labor-cost question") where descriptive prose works.
- *Framework-signaling* — term used apparently to mark framework-membership rather than do work in the sentence (judgment call — carries `?` flag).

**Ambiguous patterns:**
- *Other* — doesn't cleanly match either side; flagged for author judgment with my best read of the tradeoff.

### §3.5 Sub-pass 5 — Strategic-option test (α / β / γ)

**Purpose:** For terms surviving sub-passes 2 + 4 at Tier C or Tier D, test the three strategic options for managing the cluster.

**Three strategic options:**

- **(α) Sharpen the framework-technical subset.** New names per ratified axes (sub-pass 1) for category-(1) framework-technical cluster terms. Leave borrowed-academic uses as-is (load-bearing on established meanings). Treat descriptive uses as prose.
- **(β) Establish capitalization discipline.** *"Intergenerational X"* capitalized = framework term; *"intergenerational X"* lowercase = descriptive prose / borrowed academic concept. Same vocabulary; case-discipline does the disambiguation.
- **(γ) Reduce "intergenerational" footprint substantially.** Every category-(1) term gets a non-"intergenerational" name. The prefix is reserved exclusively for category-(3) borrowed academic uses + occasional category-(4) descriptive prose.

**Procedure:**
1. Apply each option to the cluster.
2. Test each option against M3 + M6 + M8 + M11 + M12.
3. Recommend.

**Output:** Strategic-option verdict (α / β / γ / hybrid).

### §5.2 Per-axis test

#### §5.2.1 Sense-1a (locus) and Sense-1b (mathematical role) test

**Disambiguation value:**
- Sense 1a distinguishes input-subsidy terms (atmospheric oxygen, ecosystem services, social trust) from output-externality terms (CO₂ emissions, lung tissue damage, community fragmentation). Splits cluster meaningfully.
- Sense 1b distinguishes input-variable terms (Dynastic Labor Cost, Foreclosure Cost) from output-result terms (Cost Severance, IPG, Severed Cost). Splits cluster meaningfully and cleanly.

**Orthogonality test:** Sense 1a and Sense 1b are independent. A term can be input(1a)+input(1b) (Dynastic Labor Cost is input-side cost not priced to extractor + appears as Cᵢ in RCV); input(1a)+output(1b) (Cost Severance refers to gap created by input-side under-pricing + appears as CS output of equation); output(1a)+input(1b) (Externality Tail is output-side dumped harm + appears as Cᵢ in RCV integrand). The four cells of the 2×2 are all populated.

**Verdict:** Both senses RATIFIED as distinct axes.

#### §5.2.2 Forward-vs-backward test

**Disambiguation value:** Distinguishes extraction-now-cost-later from past-extraction-residue-now. Splits cluster (most "intergenerational X" terms are forward-looking; intergenerational wealth transmission and similar are backward-looking).

**Orthogonality test:** Independent of Sense 1a/1b.

**Verdict:** RATIFIED.

#### §5.2.3 Cost gap vs subsidy gap test

**Disambiguation value:** Same fact viewed from extractor (subsidy gap = what's effectively granted) vs cost-bearer (cost gap = what's not paid). Same accounting; opposite framing.

**Orthogonality test:** Partly redundant with Sense 1a (input subsidy ≈ subsidy gap from extractor view; output externality ≈ cost gap from cost-bearer view). When tested in detail, axis 3 is the *frame* under which axis 1a is articulated — not strictly orthogonal.

**Verdict:** *RATIFIED with note* — treated as a frame-of-reference modifier on Sense 1a rather than a fully independent axis. Each term reportable in either frame; Sense 1a is the underlying axis.

#### §5.2.4 Stock-vs-flow test

**Disambiguation value:** Splits "intergenerational stocks" (Ch 9) from "intergenerational transfers" (Ch 7). A stock can be backward-accumulated; a flow can be forward-occurring.

**Orthogonality test:** Independent of forward-backward. Stock-vs-flow is about the *form* of the quantity; forward-vs-backward is about the *direction*.

**Verdict:** RATIFIED.

#### §5.2.5 Recipient class test

**Disambiguation value:** The "intergenerational" prefix conflates lineage (Dynastic Labor Cost — family/kin), descendant communities (community-disruption residue), species (intergenerational carbon harm — biosphere/all humans), nation (intergenerational fiscal architecture — political community). Distinct meanings; current vocabulary blurs them.

**Orthogonality test:** Independent of all prior axes.

**Verdict:** RATIFIED — likely high disambiguation value for the cluster.

#### §5.2.6 Substitutability test

**Disambiguation value:** Substitutability is already a framework primitive (S). The meaning of "intergenerational X" changes when substitution is possible vs impossible — substitutable resources allow forward-transfer of value (a steel substitute restores beam-strength); irreplaceable resources foreclose forever (a destroyed habitat cannot be re-grown in human-time-relevant frames).

**Orthogonality test:** Substitutability is the framework's S; partly captured in foreclosure-cost C₁ and Externality Tail E distinction. As an axis it overlaps with Sense 1b (mathematical role).

**Verdict:** *RATIFIED with caveat* — treated as a per-term substitutability annotation rather than a full disambiguation axis. Surfaces in sub-pass 4 per-occurrence audit when a term's claim depends on substitutability state.

#### §5.2.7 Reversibility test

**Disambiguation value:** ARR-relevant. Some "intergenerational X" claims hinge on reversibility (intergenerational carbon harm is largely irreversible at human-time scales; intergenerational fiscal architecture choices are reversible).

**Orthogonality test:** Partially independent of forward-backward (forward-looking + reversible = different from forward-looking + irreversible). Mostly orthogonal; some overlap with substitutability.

**Verdict:** RATIFIED — modest disambiguation value; useful for ARR-context audits.

#### §5.2.8 Direct-vs-externalized test

**Disambiguation value:** Partly overlaps Sense 1a (input subsidy ≈ direct internalized; output externality ≈ externalized). When tested in detail, this axis adds little not captured by 1a.

**Verdict:** NOT RATIFIED — collapses into Sense 1a.

#### §5.2.9 Scope of claim test (descriptive / normative / operative)

**Disambiguation value:** High. Distinguishes:
- *Descriptive* — *"the McDowell IPG was 33"* (an observation about a specific case)
- *Normative* — *"intergenerational equity requires X"* (an academic-philosophical claim)
- *Operative* — *"the Asymmetric Regret Rule directs preserve in this case"* (a decision-rule output)

**Orthogonality test:** Independent of all prior axes.

**Verdict:** RATIFIED — likely very high disambiguation value across the cluster. Often the main difference between borrowed-academic uses and framework-technical uses.

#### §5.2.10 Quantifiability test

**Disambiguation value:** Splits Europa-class (incommensurable) from McDowell-class (measurable). Modest disambiguation across the cluster — most cluster terms are in one of the two classes; few span.

**Orthogonality test:** Partly captured by Substitutability and Reversibility.

**Verdict:** *RATIFIED with caveat* — treated as a per-term annotation.

#### §5.2.11 Discount-sensitivity test

**Disambiguation value:** Distinguishes claims under DR ≈ 5% (steady-state economics convention) from DR ≈ 0 (Stern-style intergenerational ethics) from DR-independent (incommensurable claims). Applies to forward-looking terms primarily.

**Orthogonality test:** Partly overlapping with Forward-vs-backward and Quantifiability.

**Verdict:** *RATIFIED with caveat* — applies to forward-looking subset; not universally relevant.

#### §5.2.12 Legal/policy capture test

**Disambiguation value:** Distinguishes intergenerational fiscal architecture (already in policy — Social Security generational accounting) from intergenerational carbon harm (partially — carbon pricing exists but inadequately) from Dynastic Labor Cost (unpriced). Modest disambiguation.

**Orthogonality test:** Independent of prior axes.

**Verdict:** RATIFIED — useful for surfacing where framework-vs-academic-vs-prose distinctions track existing legal/policy categories.

## §9. Strategic-option test (α / β / γ)

### §9.1 Option α — Sharpen framework-technical subset only

**Definition:** New names per ratified axes (sub-pass 1) for category-(1) framework-technical cluster terms. Leave borrowed-academic uses as-is (load-bearing on established meanings). Treat descriptive uses as prose.

**M3 (book content) test:** Minimal disruption to chapter prose. Renames only the few category-(1) terms that fail Tier A/B/C — likely 4-8 terms. Most of the "intergenerational + X" proliferation stays as-is; the prefix continues to do mixed work but the framework-technical subset is sharper.

**M6 (academic) test:** Strong. New names per ratified axes will pattern-match academic conventions (regret-theory, real-options, generational accounting) clearly. Borrowed-academic uses preserve their established cite-paths.

**M8 (test-of-time) test:** Strong. Minimal vocabulary churn = minimal disruption to existing chapter-prose investment. Sharper framework subset = stronger durability.

**M11 (critic survival) test:** Adequate. Critic encountering the framework-technical subset with sharper names + ratified axes finds the vocabulary defensible. Critic encountering "intergenerational equity" alongside "intergenerational fiscal architecture" alongside (renamed) framework-technical terms might still ask "why mix?" — partially answered by the borrowed-academic-vs-framework-technical distinction.

**M12 (lineage / intellectual honesty) test:** Strong. Each renamed framework-technical term gets explicit Tier-B-or-D lineage citations.

**Effort estimate:** 4-8 renames; per-occurrence sweep of 50-150 occurrences across chapters; ~3-5 hours Phase 5 work.

### §9.2 Option β — Capitalization discipline

**Definition:** *"Intergenerational X"* capitalized = framework term; *"intergenerational X"* lowercase = descriptive prose / borrowed academic concept. Same vocabulary; case-discipline does the disambiguation.

**M3 test:** Modest disruption — every chapter occurrence audited for capitalization correctness; some sweep work but no rename work.

**M6 test:** Modest. Academic readers encountering "intergenerational equity" lowercase will read it as the established Brown-Weiss-Howarth concept; capitalized "Intergenerational Equity" would signal framework specialization. But: is *"Intergenerational Equity"* even a framework term? If not, the discipline doesn't gain anything for that phrase — the discipline only matters where the framework genuinely has a framework-technical use of the same prefix.

**M8 test:** Weak-to-modest. Capitalization discipline is fragile — production typesetting frequently overrides authorial capitalization choices; readers may not perceive the case-as-meaning convention; future editions may homogenize. Discipline depends on infrastructure that may not survive multiple editions.

**M11 test:** Weak. Critic could argue: *"You're using capitalization to do disambiguation work that real terminology should do. Pick different words."*

**M12 test:** Adequate. Lineage handling can stay unchanged.

**Effort estimate:** Per-occurrence audit + capitalization-correction sweep; ~3-4 hours Phase 5.

### §9.3 Option γ — Reduce "intergenerational" footprint substantially

**Definition:** Every category-(1) framework-technical term gets a non-"intergenerational" name. The prefix is reserved exclusively for category-(3) borrowed academic uses + occasional category-(4) descriptive prose.

**M3 test:** Substantial disruption. Renames Intergenerational Pricing Gap (Ring-2 ratified) — that's a major existing-term rename. Renames any other framework-technical "Intergenerational X" terms. Multi-chapter sweep. ~8-15 hours Phase 5.

**M6 test:** Strong. Vocabulary cleanup is decisive — readers no longer face mixed-register "intergenerational" overlap.

**M8 test:** Strong if executed; weak during transition. The transition period (chapters bearing both old + new names; supersession records; decision documents) creates audit-trail complexity.

**M11 test:** Strong. Critic finds clean vocabulary discipline without prefix overlap.

**M12 test:** Strong but heavy lift. Each renamed term needs full re-lineage.

**Effort estimate:** 6-12 renames including Ring-2 IPG rename; substantial sweep; ~10-15 hours Phase 5; rigor pass per renamed Ring-2 term.
