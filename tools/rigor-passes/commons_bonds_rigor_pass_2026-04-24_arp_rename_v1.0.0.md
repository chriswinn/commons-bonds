## §1. Executive summary

**RECOMMENDED: Option B — RENAME to "Reversibility Default" (operational-instruction primary) OR Option A — RENAME to "Asymmetric Regret Rule" (minimal-change preserving regret-theory lineage).** Both are defensible; the choice is authorial-voice-dependent. **Status quo "Asymmetric Regret Principle" is REJECTED on overclaim grounds.** The other six candidates either weaken the rule's content or fail M5 dinner-table.

**The decisive tension surfaced by the pass:**

ARP carries TWO things the rename must preserve or shed deliberately:
1. **Operational instruction** (default to reversible) — what a reader DOES with the rule.
2. **Decision-theoretic lineage** (Savage 1951 minimax-regret + Loomes-Sugden 1982 regret theory) — what the rule connects to academically.

Different rename candidates prioritize differently:

| Candidate | Operational clarity | Decision-theoretic lineage | Bidirectional capture | Overclaim |
|---|---|---|---|---|
| **Asymmetric Regret Principle** (current) | Weak — "regret" is psychology-flavored | STRONG (clean Loomes-Sugden alignment) | Implicit | "Principle" overclaims |
| **Asymmetric Regret Rule** | Same as current | STRONG | Implicit | "Rule" softer; OK |
| **Reversibility Default** | STRONGEST — "default to reversible" | WEAKER — drops "regret" framing | Implicit (what's reversible varies) | None |
| **Option-Preservation Rule** | STRONG — "preserve options" | WEAKER — option-theory-flavored | STRONGEST | None |
| **Asymmetric Rule** / **Asymmetry Rule** | Weak — too generic | Lost | None | None |
| **Irreversibility Bias** | Mid — "bias against irreversible" | Lost | Implicit | "Bias" reads as cognitive-bias not rule |
| **Regret Asymmetry Rule** | Weak | STRONG (preserves regret) | Implicit | None |

**Top 3 finalists:**

1. **Reversibility Default** — operational primacy. *"When uncertain, default to the reversible option."* Strongest M5 dinner-table; loses regret-theory framing (mitigable via Tech Appendix footnote citing Savage + Loomes-Sugden).

2. **Asymmetric Regret Rule** — minimal-change. Preserves regret-theory lineage; downgrades "Principle" → "Rule" for overclaim relief.

3. **Option-Preservation Rule** — elegant bidirectional framing. Pairs naturally with the Comet-flyby/Europa example pattern. Connects to real-options-theory family.

**My recommendation:** **Reversibility Default** for these reasons:

- **M5 dinner-table win is decisive.** Average reader gets the rule on first encounter. *"Default to the reversible option."* Operational. Visualizable. Memorable.
- **The aphorism survives intact:** *"We can always extract later. We can never un-extract."* This aphorism literally describes reversibility-default. Renaming the rule to match the aphorism's actual content is consistent.
- **Decision-theoretic lineage** doesn't require "regret" in the name. Tech Appendix footnote citing Savage 1951 + Loomes-Sugden 1982 + Lempert et al. RDM positions the rule academically without the name carrying the lineage.
- **Bidirectional flip preserved** through context-dependence of "reversible" (Comet flyby: extraction is reversible relative to losing access; Europa: preservation is reversible relative to irreversible damage).
- **No overclaim** — "Default" is operationally precise; doesn't claim foundational principle status.

**Fallback:** if Chris prefers preserving the "regret" framing for adoption-continuity (some chapter prose may already lean on "regret"), **"Asymmetric Regret Rule"** is the minimal-change alternative.
