### §1.1 Pass 3.3 acceptance protocol

Per Stage 3 template §"Pass 3.3," the acceptance test scores per-character
on the INCLUDE / NEUTRAL / EXCLUDE scale (✓✓✓ → ✓✓ → ✓ → NEUTRAL → ⚠ →
⚠⚠ → ⚠⚠⚠). Tier 1 ⚠⚠⚠ blocks ship. Aggregate verdict reports total tally
plus per-tier breakdown.

character's verdict is judged on the post-Phase-C-β state, with explicit
note of changes from the prior 2026-05-10 comparison §1.15 matrix scoring.
The three Pass 3.2 Phase C-β edits are evaluated for per-character
impact.

### §1.2 Pass 3.4 robustness protocol

Per Stage 3 template §"Pass 3.4" + audience-pressure-test-construction
guide §3, the adversarial set tests robustness. Verdict-floor is EXCLUDE
(adversarial characters are SELECTED for hostile read). The diagnostic
output is the **thread-pull synthesis table**: which threads do
adversarial characters collectively find, and are they (a) load-bearing
chapter/pitch claims that disarming moves should hold, or (b) procedural-
cosmetic flags that spot-fixes could remove?

Adversarial-set size per `audience-pressure-test-construction.md` §8
defaults for 250-500w pitch: 2-3 characters baseline. **This artifact uses
5 characters** to give substantive thread-pull synthesis depth given Pass
3.2 §10.1's explicit Public Choice / Coasean probe flagging + the
F-V-AeonC-5 V3 revision's specific cross-pass implications for adversarial
