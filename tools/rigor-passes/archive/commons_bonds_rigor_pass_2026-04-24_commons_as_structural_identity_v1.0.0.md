## §4. Two CIT sub-forms (methodology refinement output)

### §4.1 The distinction

AIT's "invert the abundance state" language collapsed two operationally distinct moves:

**Commons-Absence Inversion:** test whether the cost depends on the commons EXISTING.
- Paradigm: oxygen in space; habitability in extracted regions; ecosystem in Deepwater.
- Operational instruction: imagine the commons not existing at all.
- Cost revealed: what it would take to replace the commons.

**Commons-Consumption Inversion:** test whether the cost depends on the commons being CONSUMED (drawn down, occupied, spent).
- Paradigm: commute time consuming lifetime-hours; attention-economy consuming attention commons; forest-clearing consuming ecosystem commons.
- Operational instruction: imagine the commons not being consumed.
- Cost revealed: what the commons would be used for if it weren't being drawn down.

### §4.2 Why both are legitimate CIT sub-forms

Both invert the CURRENT STATE of the commons. The current state is either:
- **Present/intact** (commons exists + functions) — Absence-Inversion imagines removing it.
- **Being consumed** (commons exists but being drawn down) — Consumption-Inversion imagines the consumption not happening.

Each sub-form reveals costs by imagining the OPPOSITE of the commons's current relationship to extraction. The unified move is "imagine the commons-extraction relationship inverted."

### §4.3 Application guidance

For any candidate cost, identify the commons-extraction relationship:
- If extraction removes/destroys the commons → Commons-Absence Inversion.
- If extraction draws down / consumes / occupies the commons → Commons-Consumption Inversion.
- If extraction does both (e.g., mining removes habitability AND consumes the land) → apply both sub-forms; integrate costs.

### §4.4 Verdict

Both sub-forms should be named in the Tech Appendix methodology section. Ch 6 prose can introduce CIT with one paradigm case from each (McDowell Habitability for Absence; commute-time for Consumption) to establish the method's flexibility.
