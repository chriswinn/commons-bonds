## §0. Purpose and method

### 0.1 Why this document exists

The case-study audit v1.0.1 (Session A) verified which cases earn their place and where each belongs. This document verifies the chapters themselves: does each chapter's current draft-or-guidance state pass the v1.3.0 rigor battery? What chapter-level gaps exist beyond case-integration? What structural patterns across chapters reveal Book 1 strengths and weaknesses?

The case audit evaluated content-that-might-go-in-chapters. The chapter audit evaluates the-chapters-themselves.

### 0.2 Method

For each chapter, this audit:

1. Names current state (drafted / guidance-only / stub).
2. Summarizes the chapter's current load-bearing argument + register.
3. Runs the v1.3.0 rigor battery at compressed depth: Group A + §5A standing + selected Group B–F + character-suite pressure points that produce findings.
4. Delivers a verdict: **Ready for integration rewrite** / **Needs substantial revision** / **Blocking items must be resolved first** / **Draft not yet begun**.
5. Names the gap between current state and "chapter that earns its place in the completed Book 1."
6. Names cross-chapter tensions the chapter surfaces.

Cross-chapter §§3–5 then synthesize patterns.

### 0.3 Test 29 verdict for this document

**Mixed — keep together (with justification).** Per-chapter profiles are canonical reference for chapter-drafting work (book-worthy when read as drafting guidance); rigor verdicts are scaffolding (decision trail). Place at `core/chapters/` parallel to case-study and dimensions audits; retain mixed content with justification per v1.3.0 §5A.2.
