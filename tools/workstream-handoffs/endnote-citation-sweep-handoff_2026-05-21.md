## 1. Background + why this workstream exists

The chapter prose currently carries no inline citations. Trade-press publisher fact-checkers (Knopf / FSG / PublicAffairs / Penguin Press imprints) will chase any unsourced striking factual claim during pre-publication review. Citation infrastructure (endnotes or footnotes) needs to be built once.

Doing this per-chapter as Phase A audits land is wasteful — better to accumulate findings across all 10 chapters' Phase A audits + Ch 2 + Ch 3 named-voice additions + Darity bibliography expansion + Phase B whole-book audit + any additional gaps the publisher's own fact-checker flags, then run a single coordinated endnote-building sweep against a consolidated inventory.

This handoff is the consolidated inventory. Per-chapter sweep sessions fire from here.

**Decision 1: Endnote format.**
- **Option A:** Chapter-end notes inside each chapter file (`Chapter__5_TheAccountabilityGap.md` ends with a `## Notes` section + numbered references).
- **Option B:** Separate endnotes directory `manuscript/endnotes/` with one file per chapter.

- **Recommendation:** Option C (inline footnotes); footnote syntax renders cleanly through both wkhtmltopdf and Chrome PDF; keeps citations co-located with their chapter; matches trade-press convention.

**Decision 2: Citation style.**
- **Option A:** Chicago author-date.
- **Option B:** Chicago full-note + bibliography.
- **Option C:** Trade-press hybrid (Atlantic / NYRB style — short-form author + title + year inline, full citation in bibliography).
- **Recommendation:** Option C — matches Mazzucato / Christophers / Pistor / Hacker comp titles; reads as trade-press not academic.
