## Workstream scope

Build a per-comp deep matrix beyond the current Tier 1 / Tier 2 / Tier 3 categorization. For each selected comp title, populate:

| Column | Content |
|---|---|
| Title + author + year | Bibliographic anchor |
| Publisher + imprint | Where it shelved |
| Hardcover / paperback / list price | Market signal |
| Format pages + word count | Comparable-to-Commons-Bonds size |
| BookScan / Publishers Marketplace estimated trajectory | If accessible — sales arc |
| Agent (if known) | Pipeline intelligence |
| Editor / acquiring imprint editor | Pipeline intelligence |
| Reviews — top 3 outlets | Review-network signal |
| Blurbs — who blurbed | Endorsement-network map |
| Comp-comp titles — what THIS book listed as its own comps | Discovery of adjacent shelf |
| Why-this-book-is-like-it (1 sentence) | What *Commons Bonds* shares with the comp |
| Why-this-book-is-different (1 sentence) | The differentiation pitch |
| Audience overlap (Tier 1/2/3) | Who reads both? |
| Mention-in-query? (Y/N) | Is this comp going into the query letter? |
| Mention-in-proposal? (Y/N) | Is this comp going into the proposal §02? |
| Notes | Anything else load-bearing |

**Why a deep matrix:** Agents read the comp-titles section of a query letter in seconds. They want to see (a) the author has read the comps; (b) the author understands the bookshelf; (c) the book differentiates clearly. A vague list of names doesn't pass that test; a deep matrix produces query-letter sentences that are ready to deploy.

The matrix also answers downstream questions:
- Which comp's agent is the strongest agent target (cross-reference with `agent-prep-handoff_2026-05-09.md`)?
- Which comp's blurbers are reachable for *Commons Bonds* blurbs?
- Which comp's review outlets should be in the marketing plan §04?
- Which comp's editor at acquiring publisher is the strongest acquisitions target?

## Why this matters

The Aeon and Noema rigor passes both flagged comp-titles fluency as a Tier-1 signal for the agent/publisher pipeline ("Mazzucato names the extraction. The framework adds the accounting" is the kind of sentence that requires real comp-titles fluency). The book proposal §02 has the substantive list; the matrix turns the list into operational intelligence usable across the query letter, the proposal, the marketing plan, and the agent-target prioritization.

Also: comp-titles work needs to be **fact-checked against current state of the publishing market** (what's selling, what's signed for fall, who's the hot acquiring editor) and **kept current**. A matrix is easier to keep current than free prose. This is foundational infrastructure that pays off across every downstream publishing-pipeline artifact.

## Methodology

1. **Confirm the comp set.** Start from `publishing/book-proposal/02_comp-titles.md` Tier 1+2+3. The user may want to add or remove. Adjacent candidates from prior rigor passes: Mazzucato (Value of Everything + Mission Economy); Raworth (Doughnut Economics); Pistor (Code of Capital); Sandel (Tyranny of Merit); Christophers (Our Lives in Their Portfolios); Giridharadas (Winners Take All); Hickel (Less Is More); Darity & Mullen (From Here to Equality); Tooze (Crashed; Shutdown); MacAskill (What We Owe the Future); Mulgan (Future People); Susskind (Future of the Professions); Sassen (Expulsions); Patel & Moore (A History of the World in Seven Cheap Things); Rodrik (Has Globalization Gone Too Far?). Probably 10–15 actual comps in the matrix.
2. **Per comp, populate the matrix.** Web research for each: publisher, agent, editor, reviews, blurbs. Cross-reference against `research/literature/bibliography.md` for citation entries already verified.
3. **Rank for query-letter use.** Most query letters use 2–3 comps; pick the strongest 3.
4. **Rank for proposal use.** The proposal's §02 supports 5–8 comps with substantive prose treatment.
5. **Rank for agent-target prioritization.** The matrix's "Agent" column drives the `publishing/agents/_pipeline/targets.md` populating effort.

**Per-comp budget:** ~30–60 minutes of web research + matrix population. **Total estimated effort:** ~10–15 comps × ~45 min = 7–12 hours of session time.

## What NOT to do

- Do not duplicate `02_comp-titles.md` — augment with `02a_comp-titles-deep-matrix.md` as a sibling.
- Do not start populating the matrix without confirming the comp set with the user — research effort is non-trivial and shouldn't be wasted on titles that don't make the cut.
- Do not over-populate the matrix with marginally-comp titles — diminishing returns past ~15 entries.
- Do not assume Web research is always accurate — verify with multiple sources for high-stakes data (sales, agent identity).
