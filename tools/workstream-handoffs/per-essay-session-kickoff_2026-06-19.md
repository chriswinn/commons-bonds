## SHARED CONTEXT (every essay session reads this first)

**Start:** create an isolated `git worktree` (`tools/drafting-templates/worktree-isolation-paste-text.md`),
slug = your essay (e.g. `noema-draft`). Pull newest `origin/main`. Read `MAP.md` then `STATE.md`.

**DRAFTING DISCIPLINE — MINIMUM-ONLY, NO MAXIMUM (load-bearing, ratified 2026-06-19).**
Never hand a drafter a word *ceiling*. Set only a **minimum** (or "substance drives length");
a maximum dramatically hurts prose quality — the drafter compresses or omits beats to hit
the number. A venue word limit is a **later, collaborative hand-edit-out** step, then the
author's redline last. (Empirically: McDowell drafts handed a 900 max amputated ~40% of the
argument; min-only drafts ran to their natural ~1,300–1,600w.) See
`tools/memory/feedback_substance_drives_length.md` (refinement 2026-06-19). Try a couple of
different minimums if useful, to see which beats each surfaces.

**The current best version is whatever is on `origin/main`** — there is no "canonical"
(the word is retired; it masks drift). Verify content against `main`, not labels/dashboards.
A genuine improvement gets **ADDED** if the piece is not yet submitted; if it is submitted-
under-review, feature-freeze + the author may approach the editor; if published, revision.
(`tools/memory/feedback_main_is_current_best_nothing_frozen.md`.)
