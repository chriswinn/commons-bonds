## Why the workflow changed

The workstream-based architecture assumed the Option-C framework (two pillars + two methodologies) was settled and that chapter-level integration would happen in discrete bundled passes. The rigor-protocol v1.2.0 → v1.2.1 work (April 21-22) then the v1.28.0 dimension-rename session (April 22) evolved the framework architecture materially:

- **AIT became the epistemic core**; dimensions became organizational scaffolding (per C9 patch).
- **10 dimensions ratified** via v1.21.0 + Part 2 rigor pass (superseded the 7-layer framework the workstreams assumed).
- **Naming cohort in progress** — container rename "layers" → "dimensions" + individual dimension renames (Habitability + Ecosystem locked; Kinship leading for Demographic; Priority-2 cohort pending).
- **Rigor-protocol + rigor-pass-record pattern** replaces parts of the workstream-based tracking.
- **Guidance-doc-centric drafting** (chapter guidance docs as canonical planning locations) replaces workstream-based chapter-level planning.

The surviving actionable content routes into this newer architecture cleanly, per the mapping above.
