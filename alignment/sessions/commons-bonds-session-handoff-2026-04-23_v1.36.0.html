TOP PRIORITY — framework-scope decision via PCR run

v2.1.0 was built specifically for this decision. The path set is articulated at protocol §22.8:

  • Path A — Status quo. All 10 abundance dimensions + 8-tier decomposition + AIT + RCV + CS + formal vocabulary + 18 case studies + extensive elaboration. Current state.
  • Path B — Compress to CORE. AIT + RCV + CS + vocabulary + small case-study illustrative set. Dimensions and tiers retired. Aggressive simplification.
  • Path C — Hybrid (dimensions kept, tiers retired). 10 dimensions as case-classification heuristic; 8-tier retired.
  • Path D — Hybrid (tiers kept, dimensions retired). 8 tiers as accounting structure; 10 dimensions retired.
  • Path E — Aggressive minimum. CORE only + cases as illustrations of mechanism, not framework.
  • Path F — Variables-not-dimensions reframing (NEW). CORE same as Path B. 10 abundances + 8 tiers reframed as examples of variables AIT discovered in our cases — illustrative, not ontological. Framework becomes generative: readers apply method in their own contexts. Asteroid-miner-considering-off-world-contract example directly satisfies Goal-2.

The PCR run will produce:

  • CORE-survival gate verdict per path (M1 applied).
  • Standing-gate verdicts per path (3 gates × 6 paths).
  • Per-path × per-module findings for modules mapped to Chris's four goals.
  • Per-goal aggregate score per path.
  • Cross-path trade-off surface (where paths win on one goal and lose on another).
  • Per-goal recommendations + decision rationale.

After the PCR lands, the framework-scope decision becomes Chris's explicit choice with the suite's findings in front of him. The chosen-path revisions then become the next session's work: execute the revisions on framework, chapter drafts, case studies, audits, and any downstream artifacts.
