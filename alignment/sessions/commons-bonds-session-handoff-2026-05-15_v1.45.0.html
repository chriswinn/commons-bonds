Build pipeline iterations (multiple converging threads)

The Darity packet drove substantial build-pipeline work as rendering defects surfaced and got fixed in sequence:

  1. HTML → PDF engine swap: wkhtmltopdf (commits 1e984b9 + initial regenerations) → preferred-Chrome-headless with wkhtmltopdf fallback (commit cf24f57). Root cause: wkhtmltopdf 0.12.6's patched Qt 5.5 font loader cannot enumerate macOS TrueType Collections (.ttc); Georgia/Times/Charter silently substituted with Helvetica, Plane-1 chars (𝒞 U+1D49E) fall to .LastResort tofu. Chrome uses the platform-native font stack and matches in-browser render exactly.
  2. EB Garamond glyph-coverage workaround: chapter PDFs initially rendered em-dashes in bold contexts and ≈ as tofu boxes (EB Garamond 12-Bold lacks U+2014; all weights lack U+2248). Fix: tools/scripts/fallback-header.tex with fontspec + newunicodechar mapping the two codepoints to DejaVu Serif (commit d238f2c). Surrounding font context preserved for everything else.
  3. Alt-script as comparison vehicle: tools/scripts/build-derivatives-alt.sh (commit e183953) wires the fallback-header in automatically with a -H toggle. Coexists with the baseline build-derivatives.sh; convergence to a single canonical script is open work (see workstream candidates below).
  4. Documentation: tools/scripts/README.md created (commit 6f5a19c) + defect-triage section added (commit c4eb93c). README now carries the defect → pipeline mapping: HTML→PDF defects route to CSS / wkhtmltopdf-or-Chrome layer; MD→PDF defects route to xelatex font coverage / header overrides; any-source→DOCX defects route to reference.docx styles.
  5. .gitignore: missfont.log ignored (commit 8caeed3) so xelatex's transient build artifacts don't pollute the working tree.
