4. Batched roadmap

Each batch is sized for a single session and ordered by risk (lowest first). Batches 1–3 are safe reorganization and format work. Batches 4–5 involve content decisions. Batch 6 is optional polish. Batch 7 is the archive triage.

Batch 1 — Fix misplaced files

Effort: small. Risk: low. No content changes.

  1. Diff alignment/decisions/session-handoff-2026-04-19.md vs. alignment/sessions/commons-bonds-session-handoff-2026-04-19.md. If identical or the decisions/ copy is a subset, remove the decisions/ copy. If it has unique content, merge then remove.
  2. Move alignment/decisions/eight-tier-decomposition-v10.md to core/decomposition/ beside the HTML.
  3. Decide whether to move the two tools/ HTML files to research/validation/ and research/methodology/ respectively (create research/validation/ if so). Alternatively, keep them in tools/ and update the design doc to match reality.
  4. Rename publishing/{targets}/ → publishing/targets/.

Claude's role: produce a diff report for the two session-handoff files so Chris can decide. Produce a single "cleanup actions" text file listing every move/rename to execute by hand.

Batch 2 — Resolve chapter version duplicates

Effort: medium. Risk: low (content review only). Sets up Batch 4.

  1. For each chapter with multiple versions (Ch. 5, Ch. 6, Ch. 9, Ch. 10), Claude reads each version and produces a short comparison memo: length, structure, evident recency, which is the working copy.
  2. Chris picks canonical versions. Non-canonical versions get moved to archive/chapter-drafts/.
  3. Same pass for the two exact-duplicate "consider including" HTML files (9 vs. 12) and the prototype .docx/.html pair.

Claude's role: produce a single "chapter-version-comparison.html" report covering all affected chapters in one pass, plus the dedupe recommendations.

Batch 3 — Convert workstreams 08, 11–17 from HTML to MD

Effort: medium. Risk: low (1:1 format conversion, no content interpretation).

  1. Claude fetches each of the 8 HTML files, converts to well-formed Markdown with preserved structure (headings, lists, code, links).
  2. Output as 8 local .md files for Chris to upload.
  3. Chris uploads, then deletes the .html originals.

Claude's role: generate all 8 .md files in one or two rounds, each verified by re-reading before handing over. Zip for single-file download if count is awkward.

Batch 4 — Convert manuscript chapters (Part A: Ch. 1–5 + Dedication + Author's Note)

Effort: large. Risk: medium (content fidelity matters). This is the book.

  1. After Batch 2 resolves duplicates, convert each canonical chapter docx → .md (narrative chapters) or .html (Ch. 6, math-heavy — already has an .html version to reconcile).
  2. Each conversion is handed off as a local file. Chris verifies length and content before upload.
  3. Chris uploads to manuscript/chapters/ and moves originals to archive/docx-source/ (not delete — keep as fallback).

Claude's role: convert one chapter per request OR batch 2–3 at a time depending on chapter length. Always echo back file size after creation so Chris can sanity-check against the source docx size.
