Commons Bonds operating core (canonical — update only on protocol revision)

AIT positioning

The Abundance Interference Taxonomy is the framework's epistemic core. Layers are organizational scaffolding, not load-bearing structure. Layer additions, mergers, or renames are organizational revisions, not framework-integrity events. Canonical statement: C9 patch §3. Note (v1.24.0): the abundance-layer set that the AIT operates on (distinct from AIT layers in the organizational sense) has not been rigor-validated as of this handoff. See critical callout above.

Two-path rigor test — apply before ratifying any architectural claim

  • Path 1 — Allocation symmetry: Is the structural value-capture / cost-bearing asymmetry present?
  • Path 2 — Shield absence: Is the distance condition (the shielding mechanism) absent?

Both paths must confirm. Structural allocation is the mechanism of cost severance. Distance is the shielding condition.

Merger gate — apply before ratifying any new or renamed layer

Demonstrate bilateral independence with cases in both directions: a case where Layer A costs occur without Layer B costs, and vice versa. If independence fails in either direction, merger is indicated.

Primitiveness gate — apply before ratifying any new layer

The layer's revealed costs must not decompose cleanly into compositions of existing layers. If they do, the candidate is a cross-layer concept, not a new layer.

Operating discipline

"Push harder so we only rewrite once." Applies to layer set, protocol, AIT positioning, chapter framing, book scope, and now the abundance-layer set itself. The recursive-validation gap surfaced this session is an instance of the discipline being applied: better to catch and fix now than after the layers propagate further through ratified content.
