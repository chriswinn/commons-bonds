Pipeline reference (proven in Batch 2)

Same as v1.0.0 Appendix, with one clarification added from Batch 2 experience (see step 2).

  1. Download via Google Drive:download_file_content (not read_file_content — the latter returns empty for raw HTML). The blob arrives at embeddedResource.contents.blob as base64.
  2. If the ID returns "Item not found," search by title using Google Drive:search_files with a query like title contains 'distinctive_substring'. Parent folder is not guaranteed to be stable — files may have moved between when the v1.0.0 handoff was written and now.
  3. Save each blob to /home/claude/commons-bonds-raw/{name}.b64.
  4. Decode + preprocess + convert: 
    import base64, re
from bs4 import BeautifulSoup
from markdownify import markdownify

html = base64.b64decode(blob).decode("utf-8")
soup = BeautifulSoup(html, "html.parser")
for tag in soup.find_all(["style", "script"]):
    tag.decompose()           # .decompose() removes contents too — NOT strip=
md = markdownify(str(soup), heading_style="ATX", bullets="-")
md = re.sub(r"\n{3,}", "\n\n\n", md).strip() + "\n"   # collapse >2 blank lines
  5. For files with heavy math/formulas: skip the markdown conversion; write the decoded HTML directly to {name}.html.
  6. Output destination: /mnt/user-data/outputs/commons-bonds/.
  7. Spot-check before presenting:
    • Grep each .md for font-family and body { — must return zero.
    • Count ^| lines — tables should be preserved.
    • For .html outputs, verify DOCTYPE and closing </html> present.
  8. Call present_files on every output plus the new handoff, then stop. Chris does the GitHub push.
