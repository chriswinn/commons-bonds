## §2. The informal CIT + informal RCV two-step (decision-time methodology)

The framework's formal apparatus (Tech Appendix Theorem-set + RCV integral + Cᵢ admission via Four Gates) is architectural — it makes the framework rigorous. At decision-time, the reader doesn't need the formal apparatus; they need a usable two-step.

**Step 1 — Informal CIT (Commons Inversion Test).**
Ask whether the cost of the decision is *scarcity-grounded*. The counterfactual inversion:
- Would this cost exist if the relevant commons were inexhaustible? (CCI — Counterfactual Commons Inversion)
- Would this cost be priced differently if the commons were absent? (CAI — Counterfactual Abundance Inversion)
- If yes to either: the cost is scarcity-grounded; the framework illuminates it. If no: this is a pricing question, not a cost-severance question.

**Step 2 — Informal RCV (Residual Commons Value).**
Mentally compute the cost over the affected horizon:
- Time per unit × units per period × periods affected = cost
- Discount rate optional at the informal level
- Apply Externality Tail intuition: does this cost persist after the decision is reversible? If yes, the tail magnifies the integrated cost.

**Worked example — the commute trade (Personal Stories Candidate #1):**

Apartment manager presents two options:
- Option A: closer apartment, $X/month higher rent
- Option B: farther apartment, $X/month lower rent + 30-min commute each way

Naive analysis (price-only): rent differential is the cost.

Informal CIT: is the commute time scarcity-grounded? Would the cost of the commute exist if the relevant commons (time; cognitive bandwidth; physical proximity) were inexhaustible? **Yes.** Time is irreducibly scarce; the commute consumes a non-renewable daily resource.

Informal RCV: 30 min × 2 (each way) × ~250 commute-days/year × N years = ~250 hours/year × N years. At reasonable hourly self-valuation, the integrated cost dwarfs the rent differential. The cost is *severed* from the rent quote — the manager isn't pricing it; the lease isn't pricing it; the prospective tenant must price it themselves or eat it.

**This is the framework's decision-time affordance:** the reader, equipped with the informal two-step, surfaces costs that the existing pricing apparatus doesn't surface.

## §3. Literature-pattern analysis — three structural patterns in comparable framework books

How have similar books handled the structural question of decision-time tooling vs ex-post analysis?

### §3.1 Pattern 1 — Decision-time FOREGROUNDED

**Exemplar:** *Nudge* (Thaler & Sunstein, 2008). Framework (choice architecture / libertarian paternalism) gets ~80 pages; the rest is applied + decision-time how-to (saving, healthcare, marriage, organ donation).

**Other examples:** *Predictably Irrational* (Ariely, 2008).

**Why it worked:**
1. **Thin framework, applied weight.** The framework is simple enough that the applied work doesn't strain it. The framework's value IS the applications.
2. **Institutional shield for prescription.** Sunstein had Harvard Law + later OIRA. Ariely had Duke + decades of behavioral-econ research. Prescribing "do X" needs cover.
3. **Audience appetite.** By 2008, behavioral econ had built trade-press readiness for applied behavioral books.

**Why this pattern is risky for Commons Bonds:** the framework is *thick* (CS = RCV − B; 7 Ring-1 + 13 Ring-2 terms; 5 theorems; Cᵢ apparatus; Path F + Four Gates). A *Nudge*-style applied weight would strain it. Author shield is being-the-source-of-the-framework + Berggruen/Noema track + nursing career — closer to Raworth than Sunstein. Insight #13 already flagged this as a failure mode.

### §3.2 Pattern 2 — Decision-time THREADED

**Exemplar:** *Doughnut Economics* (Raworth, 2017). Framework (the doughnut + 7 ways) + chapter-level applied threading. No "how to apply" chapter. Each chapter ends with concrete examples + practitioner notes that show the framework getting *used*. Hugely successful — Amsterdam adopted Doughnut as municipal policy framework (2020); Brussels, Copenhagen, Portland followed.

**Other examples:**
- ***Mission Economy*** (Mazzucato, 2021) — framework (mission-oriented innovation) + Apollo case + applied threading. Methodology emerges from case treatment, not a methodology chapter.
- ***The Value of Everything*** (Mazzucato, 2018) — same threading model.
- ***Mine!*** (Heller, 2021) and ***The Gridlock Economy*** (Heller, 2008) — framework (anticommons / ownership-as-bundle) + domain chapters (real estate, online subscriptions, biotech). Chapters do double duty: case demonstration + decision-time framing for that domain.
- ***Thinking, Fast and Slow*** (Kahneman, 2011) — framework + decision-time integrated per chapter.

**Why this pattern works:**
1. **Trust the reader to extract the methodology.** Readers see the framework get used across cases; they internalize how to use it from observation.
2. **Native register-fit.** The framework + chapter-walkthrough format reinforces "this is how the framework illuminates this domain" — naturally extends to "this is how the framework can illuminate your decisions."
3. **Compatible with epistemic-humility discipline.** Raworth's tone in *Doughnut*: "here's what the framework reveals; here's what's been tried; here's what's working in practice." Same discipline as Insight #14's Norway treatment for Accountability Bond.
4. **Institutional shield matches book scope.** Raworth at Oxford/Cambridge; Mazzucato at UCL; Heller at Columbia Law. Institutional but not coauthored-advocacy — same shield class Commons Bonds has.

### §3.3 Pattern 3 — Ex-post ONLY

**Exemplar:** *Capital in the Twenty-First Century* (Piketty, 2014). Diagnostic-heavy; light policy gestures (global wealth tax) at end. No decision-time tooling.

**Other examples:**
- ***Governing the Commons*** (Ostrom, 1990) — framework (8 design principles) + cases. No how-to chapter. Won the Nobel.
- ***The Great Transformation*** (Polanyi, 1944) — framework (double movement; embedded/disembedded markets) + historical narrative.
- ***Debt: The First 5,000 Years*** (Graeber, 2011) — diagnostic framework + historical sweep.

**Why this pattern works:**
1. **The diagnostic IS the contribution.** Readers don't expect decision-time tooling.
2. **Successors do applied work.** Ostrom's principles got operationalized by Hess + practitioners; Piketty's diagnostic generated a flood of applied + policy work.
3. **Academic-monograph register tolerates it better.** Pure academic primary; trade adoption was downstream.

**Why this pattern is suboptimal for Commons Bonds:** the audience choice is academic-trade *hybrid* + supplementary policy-practitioner per Insight #25 — that's not Capital-in-21st-Century's audience (pure-academic-with-trade-spillover) and it's not Governing-the-Commons's audience (pure-academic). Commons Bonds is closer to Raworth / Mazzucato / Anderson register, where a degree of practical reach is part of the contribution.

### §3.4 Pattern-fit verdict for Commons Bonds (external layer)

| Dimension | Commons Bonds reality | Pattern fit |
|---|---|---|
| Framework thickness | THICK (5 theorems + 7 + 13 terms + Cᵢ) | Strains Pattern 1; native to Pattern 2 + 3 |
| Author shield | Source-of-framework + Berggruen/Noema + nursing career | Closer to Raworth/Mazzucato (Pattern 2); not Sunstein (Pattern 1); not pure academic monograph (Pattern 3) |
| Audience choice (Insight #25) | Academic-trade hybrid + supplementary policy-practitioner | Native to Pattern 2 |
| Epistemic-humility discipline (Insight #14) | "Demonstrate the affordance; don't codify it" | Native to Pattern 2 |
| Book-scope discipline (Insight #13) | Book 1 = framework; Book 2 = applied advocacy | Pattern 2 fits Book 1; Pattern 1's how-to register sits closer to Book 2 |
| Personal stories already in scope | Ch 1 commute trade; Ch 3 waterman (planned) | Native ingredients for Pattern 2 |

**Pattern 2 (threaded) is the load-bearing fit on every dimension.** Pattern 1 strains framework thickness + author shield + Insight #13 register. Pattern 3 underclaims practical reach for the chosen audience.
