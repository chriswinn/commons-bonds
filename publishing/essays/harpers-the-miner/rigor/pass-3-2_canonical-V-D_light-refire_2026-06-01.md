## §8. Methodology note (light-refire-scope)

A *light re-fire* of Pass 3.2 in the automatic-on-edit cascade verifies that a post-Pass-3.2 prose edit (in this case the V-D hybridization additions) has not silently re-introduced any of the Pass-3.2-cut patterns. It does NOT re-run the full Pass 3.2 audit from scratch (which would re-do the 18 → 10 em-dash audit + 7 → 0 meta-commentary audit + 15 → 1 hedge audit + rule-of-five anaphora restructure). Light re-fire scope is:

1. **Diagnostic-count verification:** confirm post-edit counts match the prior ratified target counts;
2. **Per-site audit of post-edit additions:** confirm no new em-dash / meta-commentary / hedge / rule-of-three at each addition site;
3. **Em-dash per-instance active-justification re-audit:** confirm each surviving em-dash still meets the active-justification standard (no em-dash should drift into "kept by inertia" status);
4. **Cross-pass-interaction note:** confirm no other-pass-introduced surface escapes the cut discipline.

If any of (1) through (4) surfaces a finding, the light re-fire promotes to a full per-finding-format spot-fix recommendation in Pass 3.2's Amendment-C interactive-ratification format. CLEAN verdict (this audit) means none of (1) through (4) surfaced a finding; no Phase C application required; the canonical V-D state on `origin/main` `0045952` holds Pass 3.2 RATIFIED 2026-05-27 baseline.
